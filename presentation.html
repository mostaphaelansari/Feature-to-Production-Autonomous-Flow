The Problem

Code reviews don't scale

⏳

Bottleneck

Senior engineers spend 30%+ of their time reviewing code. MRs queue up, blocking deploys for hours or days.

🎯

Inconsistency

Tired reviewers miss auth bugs on Friday afternoon. Risk assessment varies wildly between team members.

🔓

Blind Spots

No one catches when a validation check is quietly removed. Security-sensitive changes slip through.

The Solution

Diff-first autonomous review

Every MR triggers a 3-stage pipeline that understands what changed, scores the risk, and enforces policy — automatically.

Trust Model

Hardened against self-tampering

🔒

POLICY_REF

Gate scripts are checked out from a protected branch that MR authors can't push to. Even if they modify the code, the pipeline runs trusted versions.

✅

Fail-Closed

If the validator dependency is missing, the pipeline crashes — it never silently skips validation. Security is the default, not opt-in.

📋

JSON Contracts

Every stage boundary validates payloads against strict JSON schemas. Malformed data between stages is caught before it can affect decisions.

🧬

Signal Accuracy

validation_removed only fires on removed lines — adding validation code never triggers a false positive.

Architecture

Contract-driven pipeline

📄

semantic.schema.json

MR metadata, files changed, change types, risk areas, signals

⚡

risk.schema.json

Score, level, threshold bucket, weighted reason codes, actions

🎯

action.schema.json

Decision (approved/blocked), typed executable actions + params
